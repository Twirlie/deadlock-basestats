# Deadlock Hero Base Stats Comparison Tool

A web-based tool for exploring and comparing hero base statistics in [Deadlock](https://www.deadlock.game/), the third-person MOBA by Valve. Browse all heroes, search by name, compare stats, view ranking tiers, and discover how heroes scale with progression.

**Live Demo**: [Coming soon]

---

## Features

### Current
- **Hero Browser** — Grid view of all 61 Deadlock heroes with base statistics
- **Base Stats Display** — View complete starting stats for each hero
- **Responsive Design** — Works on desktop, tablet, and mobile

### Planned
- **Fuzzy Search** — Filter heroes by name
- **Stat Tierlist** — Rank all heroes by a selected stat (e.g., who has the highest max health?)
- **Hero Comparison** — Select 2 heroes and compare stats side-by-side with highlighted differences
- **Boon Progression** — View how stats scale with leveling/boons
- **Visual Enhancements** — Color-coded stats (Weapon/Vitality/Spirit), hero portraits, icons
- **Live Data** — Stays up-to-date with Deadlock patches via API

## Quick Start

### Prerequisites
- **Node.js**: v24 LTS or later
- **pnpm**: v10+ (or use `npm` if preferred)

### Installation

```bash
# Clone the repository
git clone <repo-url>
cd deadlock-basestats

# Install dependencies
pnpm install

# Extract and prepare hero data
pnpm extract

# Start the development server
pnpm dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.

## Data Source

Hero data is sourced from the [Deadlock API](https://deadlock-api.com/). Currently using a static snapshot (`heroes.json`), with plans to implement live fetching to stay current with game updates.

The `pnpm extract` command:
1. Reads `heroes.json`
2. Extracts hero `id`, `name`, and `starting_stats` for each hero
3. Writes pruned data to `data/heroes-stats.json`

## Game Mechanics Reference

Deadlock features three core stat categories:
- **Weapon** — Gun performance (Bullet Damage, Fire Rate, Ammo, DPS)
- **Vitality** — Survivability (Max Health, Health Regen, Resistances, Move Speed)
- **Spirit** — Ability effectiveness (Spirit Power, ability duration, range)

See [deadlock.md](deadlock.md) for complete game context and mechanics.

### Style Guide
- **Components**: Use TypeScript in Svelte script blocks
- **Styling**: Prefer Tailwind classes; custom CSS in `<style>` blocks when needed
- **Types**: Define props as TypeScript interfaces
- **Components**: Props should be typed (avoid `any`)

### Hot Reload Development
The dev server features Vite's Hot Module Replacement (HMR). Changes to `.svelte` files and CSS are reflected instantly without full page reloads.

```bash
pnpm dev
# Edit src/App.svelte or any component
# Changes appear in browser automatically
```
